# 모임 이벤트 관리 웹 MVP PRD

## 핵심 정보

**목적**: 소규모 모임(5~15명)의 참여 취합, 정산, 카풀 배정을 링크 하나로 해결하여 주최자의 반복 작업을 제거한다
**사용자**: 수영/헬스/친구 모임 등 비정기 소규모 이벤트를 직접 주최하거나 참여하는 일반인

---

## 사용자 여정

### 경로 A — 주최자

```
1. 랜딩 페이지
   ↓ "시작하기" 클릭

2. 로그인 페이지
   ↓ Google OAuth 로그인 완료

3. 대시보드
   ↓ "이벤트 만들기" 버튼 클릭

4. 이벤트 생성 페이지
   ↓ 이벤트 정보 입력 → 생성 완료

5. 이벤트 관리 페이지 (자동 이동)
   ↓ 링크 복사 또는 카카오톡 공유

6. 이벤트 관리 페이지
   → 참여 현황 실시간 확인
   → 미응답자 독촉 메시지 복사 (Phase 2)
   → 정산 페이지 이동 (Phase 2)
   → 카풀 페이지 이동 (Phase 3)
```

### 경로 B — 참여자

```
1. 카카오톡/문자 링크 수신 → 클릭

2. 이벤트 상세 페이지 접근
   ↓ 비로그인 상태 감지

   [이미 로그인] → 이벤트 상세 페이지 바로 진입
   [비로그인]    → 로그인 페이지로 리디렉션
                   ↓ Google OAuth 로그인
                   → 이벤트 상세 페이지로 자동 복귀

3. 이벤트 상세 페이지
   ↓ 참여 / 불참 / 미정 선택

4. 완료 (응답 즉시 반영, 페이지 유지)
   → 카풀 신청 (Phase 3)
```

## 메뉴 구조

```
[비로그인 상태]
├── 랜딩 페이지          — 서비스 소개 + 시작하기
└── 로그인 페이지        — F011 (Google OAuth)

[로그인 후 공통]
├── 대시보드             — F012 (내 이벤트 목록)
├── 이벤트 만들기        — F001 (이벤트 생성)
└── 로그아웃             — F011

[이벤트 상세 — 참여자 뷰]
├── 이벤트 정보 + 참여 응답  — F002, F003, F004, F006
└── 카풀                     — F008, F009

[이벤트 관리 — 주최자 전용 탭/링크 추가]
├── 이벤트 관리 페이지   — F003, F004, F006, F007
├── 정산                 — F005
└── 카풀 배정            — F010
```

## 페이지별 상세 기능

> **구현 기능:** `F011` | **접근:** 비로그인 포함 전체 공개

| 항목            | 내용                                                                                                                                                                  |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 서비스 소개 및 첫 진입점                                                                                                                                              |
| **진입 경로**   | 직접 URL 접근, 비로그인 상태에서 보호된 페이지 접근 시 리디렉션                                                                                                       |
| **사용자 행동** | 서비스 소개 확인 후 "시작하기" 버튼으로 로그인 이동                                                                                                                   |
| **주요 기능**   | - 서비스 핵심 가치 문구 표시 ("링크 하나로 10초 만에 참여 의사 표시")<br>- 주요 기능 3가지 소개 (참여 응답, 정산, 카풀)<br>- **"시작하기"** 버튼 → 로그인 페이지 이동 |
| **다음 이동**   | "시작하기" 클릭 → 로그인 페이지                                                                                                                                       |

| **진입 경로**   | 로그인 완료 후 자동 이동, 헤더 로고/대시보드 메뉴 클릭                                                                                                                                                                                                            |
| **사용자 행동** | 주최/참여 이벤트 목록 탐색, 새 이벤트 만들기                                                                                                                                                                                                                      |

| **주요 기능**   | - 실시간 참여 현황: 참여/불참/미정/미응답 카운트 + 응답자 이름 목록 (F003)<br>- **"링크 복사"** + **"카카오톡 공유"** 버튼 (F004)<br>- 공지사항 작성·수정·삭제 폼 (F006)<br>- 미응답자 명단 자동 표시 + **"독촉 메시지 복사"** 버튼 (F007)<br>- **"정산 관리"** 버튼 → 정산 페이지 이동<br>- **"카풀 배정"** 버튼 → 카풀 페이지 이동<br>- 카풀 탑승자 배정 확정 (F010, Phase 3) |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) — 풀스택 React 프레임워크
- **TypeScript 5.6+** — 타입 안전성
- **React 19** — UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** — 유틸리티 CSS (설정 파일 없는 새로운 엔진)
- **shadcn/ui** — 고품질 접근성 컴포넌트
- **Lucide React** — 아이콘

### 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod** — 스키마 기반 유효성 검증

### 백엔드 & 데이터베이스

- **Supabase** — BaaS (인증 + PostgreSQL + Realtime)
  - Auth: Google OAuth (이미 구현됨)
  - Database: PostgreSQL + Row Level Security
  - Realtime: 참여 현황 실시간 구독

### 서드파티

- **Kakao SDK** — 카카오톡 공유 (링크 공유 기능)

### 배포

- **Vercel** — Next.js 최적화 배포

### 패키지 관리

- **npm** — 의존성 관리
